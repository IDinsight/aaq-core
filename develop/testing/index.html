Writing and running tests

If you are writing new features, you should also add unit tests. Tests are under core_backend/tests.

Running unit tests

You need to have installed Docker

Don't run pytest directly

Unless you have updated your environment variables and started a testing instance of postrges and qdrant, the tests will end up writing to your dev environment

Run tests using:

make tests

This target starts up new postgres and qdrant containers for testing. It also sets the correct environment variables, runs pytest, and then destroys the containers.

Debugging unit tests

Before debugging, run make setup-test-dbs within core_backend to launch new postgres and qdrant containers for testing and set the correct environment variables.

After debugging, clean up the testing resources by calling make teardown-test-dbs.

Configs for Visual Studio Code

.vscode/launch.json

Add the following configuration to your .vscode/launch.json file to set environment variables for debugging:

{
    "version": "0.2.0",
    "configurations": [
        {  // configuration for debugging
            "name": "Python: Tests in current file",
            "purpose": ["debug-test"],
            "type": "python",
            "request": "launch",
            "program": "${file}",
            "args": ["--color=yes"],
            "envFile": "${workspaceFolder}/core_backend/tests/api/test.env",
            "console": "integratedTerminal",
            "justMyCode": false
        }
    ]
}
.vscode/settings.json

Add the following configuration to .vscode/settings.json to set the correct pytest working directory and environment variables:

{
    "python.testing.cwd": "${workspaceFolder}/core_backend",
    "python.testing.pytestArgs": [
        "tests",
        "--rootdir=${workspaceFolder}/core_backend"
    ],
    "python.testing.unittestEnabled": false,
    "python.testing.pytestEnabled": true,
    "python.envFile": "${workspaceFolder}/core_backend/tests/api/test.env"
}

Calling endpoints

Run the app

  • Option 1 [Docker Compose]: See Quick Start for how to start the application stack using docker-compose.
  • Option 2 [Dev Setup]: Run each container manually as per dev setup.

Call the endpoints

You can use FastAPI's Swagger UI to test out the desired endpoints

https://[DOMAIN]/api/docs # (1)!
  1. if you are using the dev setup, you can access it at http://localhost:8000/docs
Authorising the /embedding_search and /llm-response endpoints

To use these endpoints, you'll have to Authorise and set the bearer token to the value of QUESTION_ANSWER_SECRET.

The default value can be found in core_backend/app/configs/app_config.py but may have been overridden if the environment variable was manually set differently.
